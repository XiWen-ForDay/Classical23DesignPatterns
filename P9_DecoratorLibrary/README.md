装饰器模式简单说，就是在不改变原有对象的基础上，动态地给它添加一些额外功能。
它就像给一个东西不断地套上不同的外壳，每个外壳都能带来新的功能，但里面的核心东西没变。
比如你有一杯咖啡，想加牛奶就套上牛奶装饰器，想再加糖就再套上糖装饰器，这样既灵活又不会影响咖啡本身。
在代码里，通常会定义一个抽象装饰类，继承自被装饰对象的接口，然后在具体装饰类里实现额外功能。
你之前不是联想到通讯协议的分层吗？其实那就是装饰器模式的一个很好的实际应用。
