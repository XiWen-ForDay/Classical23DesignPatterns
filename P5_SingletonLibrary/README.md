单例模式最佳实践示例

豆包示例原型：

单例模式，简单说就是保证一个类在整个程序运行期间，只有一个实例，而且自行实例化并向全局提供这个实例。
它就像工控系统里的配置管理器，整个系统只需要一个统一的配置入口，所有模块都从这里获取配置，这样既能保证配置一致，又能避免重复加载资源。
比如工控上位机里的串口管理器。
整个上位机可能有多个模块需要使用串口，比如数据采集模块、设备控制模块，
但串口物理上只有一个，如果每个模块都自己创建串口对象，就会导致端口冲突。
这时候用单例模式，就能保证整个系统只有一个串口管理器实例，所有模块都通过这个实例来操作串口，避免冲突。
而且这个实例在程序启动时创建，全局都能访问，非常方便。
