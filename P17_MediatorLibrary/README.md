中介者模式最佳实践示例


豆包示例原型：

中介者模式，简单说就是定义一个中介对象，让原本需要直接相互通信的多个对象，改为通过中介者来间接通信。
这样可以减少对象之间的直接耦合，让系统更容易维护和扩展。
 
比如你做工控上位机开发，可能会遇到多个设备之间需要协同工作的情况。
比如PLC、传感器、报警器、执行器这几个设备，如果它们直接相互通信，
当需要新增一个设备或者修改某个设备的逻辑时，可能要改动很多其他设备的代码，非常麻烦。
这时候就可以引入一个中介者，比如叫“设备协调器”，所有设备都只和这个协调器通信。
传感器检测到温度过高，就告诉协调器；协调器再通知PLC启动风扇，同时让报警器发出警报。
这样一来，每个设备只需要关注自己的核心功能，不用关心其他设备的存在，耦合度就大大降低了。
