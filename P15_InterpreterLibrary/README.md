解释器模式最佳实践示例

豆包示例原型：

解释器模式是一种行为型设计模式，它定义了一种语言的文法规则，并构建一个解释器来解释该语言中的句子。
比如，像数学表达式、正则表达式、SQL语法这些，都可以用解释器模式来解析。
它的核心思想，就是把复杂的语法规则拆分成简单的“原子规则”，再通过组合这些原子规则来解析整个句子。
比如在工控系统里，经常需要解析设备发送的指令，像“启动电机A，转速1000转，持续5分钟”。
用解释器模式的话，可以把“启动”“转速”“持续”这些关键词定义为终结符表达式，然后组合成非终结符表达式，
比如“启动指令=设备+动作+参数”。
这样一来，不管设备发来什么指令，都能通过解释器解析成具体的操作，而且如果以后要加新指令，直接扩展表达式就行，不用修改原有代码。
